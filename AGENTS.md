# Stock Monitor Project Development Rules

## 1. Routing & Navigation 纪律
* **防激进缓存与假死**：对于涉及页面核心数据切换（如：从自选列表跳回首页、在搜索框中搜索新股票代码）的跳转，**必须使用 `window.location.href` 替代 `router.push`**。这能强制刷新页面，重置 React 状态并彻底击穿 Next.js 极度激进的客户端路由缓存（Client-side Router Cache），防止页面固定在旧股票或彻底假死。
* **默认首页股票**：首页默认加载股票代码为 `000725` (京东方A)。

## 2. 数据获取与缓存管理
* **防数据滞后与 Vercel 缓存**：所有客户端 fetch 请求必须添加时间戳以避免浏览器与 CDN 缓存：
  * URL 中添加 `_t=${Date.now()}` 缓存拦截参数。
  * `fetch` 选项中添加 `{ cache: 'no-store' }`。
* **增量加载**：图表周期的切换（如日 K、周 K、月 K 切换）**仅限更新图表组件本身的数据**，决不允许引起全页面行情、新闻和关联股票的重新加载或闪烁。

## 3. 前后端代码对齐
* **代码前缀纠偏防死循环**：
  * 后端接口通常会补充 `sh` / `sz` / `hk` / `bj` 前缀。
  * 前端检测到返回的 code 与当前 URL 参数不一致时，若只是多出了前缀，**决不能强行 `router.push` 刷新 URL**，以防止引起无限刷新和反复渲染的死循环。
* **自选列表管理**：添加/移除自选时，必须同时记录 `stockCode` 与 `stockName`，防止在网络延迟或缓存存在时导致加入自选后显示名称为空或代码丢失。

## 4. UI 与 ECharts 图表性能优化
* **防重新初始化卡顿**：在 React 中使用 `ReactECharts` 组件时，**不要设置 `notMerge={true}`**。这会强制销毁整个 ECharts 实例并从零重新构建，引发极明显的性能卡顿。应使用 ECharts 自带的合并差量更新。
